# Research Workflow Router

Use this placeholder skill when a DevNexus Research project needs stage
selection without loading a complete research suite.

## Workflow

1. Identify the current research stage: scope, research, plan, draft support,
   integrity check, review, revise, finalize, or process summary.
2. Read the declared research artifacts before making claims.
3. Ask for a human decision when the research question, authorship,
   methodology, citation policy, venue policy, or final submission status is
   unclear.
4. Record unsupported claims as gaps instead of smoothing over missing
   evidence.
5. Keep optional external tools behind setup checks and use the no-network
   baseline when those tools are absent.

## Boundaries

- Do not fabricate citations, source metadata, or results.
- Do not mutate source manifests or bibliography files unless the user or work
  item explicitly asks for that stage.
- Do not treat this plugin as an implementation supervisor. DevNexus core still
  owns project, tracker, worktree, coordination, and target-cycle behavior.
